## Advanced Search and Filtering

PORMAZION's search goes beyond basic keywords. You can filter by tags, dates, authors, and sections. Use the search bar at the top of your dashboard to query across all documents.

<Tabs>
  <Tab title="Basic Search" icon="search">
    Enter keywords like `API integration` to find relevant pages.
  </Tab>
  <Tab title="Advanced Filters" icon="filter">
    Combine filters:
    
    - Tags: `feature` or `bugfix`
    - Date range: Last 30 days
    - Author: Specific team member
  </Tab>
</Tabs>

<Callout kind="tip">
  Pro tip: Use quotes for exact phrases, like `"user authentication"`, and boolean operators such as `AND`, `OR`.
</Callout>

## Using Templates for Consistent Docs

Templates ensure every new document follows your standards. Create them in the Templates section of your workspace.

<CodeGroup tabs="YAML,JSON">
  ```yaml
  title: "API Reference"
  sections:
    - name: "Endpoint"
      fields:
        - path: method
          type: string
        - path: url
          type: string
  sidebar: true
  ```
  ```json
  {
    "title": "API Reference",
    "sections": [
      {
        "name": "Endpoint",
        "fields": [
          {"path": "method", "type": "string"},
          {"path": "url", "type": "string"}
        ]
      }
    ],
    "sidebar": true
  }
  ```
</CodeGroup>

To apply a template, select it when creating a new page.

## Setting Up Team Permissions

Manage who can view, edit, or admin your docs with granular controls.

<Steps>
  <Step title="Navigate to Settings" icon="settings">
    Go to Workspace `{settings}` > `{permissions}`.
  </Step>
  <Step title="Add Users" icon="users">
    Invite team members by email.
  </Step>
  <Step title="Assign Roles" icon="shield">
    Choose from Viewer, Editor, or Admin.
    
    ```
    Viewer: Read only
    Editor: Edit content
    Admin: Manage users and settings
    ```
  </Step>
  <Step title="Apply to Sections" icon="folder">
    Set permissions per folder or page.
  </Step>
</Steps>

## Customizing Layouts and Branding

Tailor your documentation to match your brand. Update colors, logos, and layouts in the Branding panel.

<Expandable title="Advanced CSS Customization" default-open="false">

  ```css
  :root {
    --primary-color: #3B82F6;
  }
  
  .sidebar {
    background: var(--primary-color);
  }
  ```
  
  Preview changes live before saving.
</Expandable>
